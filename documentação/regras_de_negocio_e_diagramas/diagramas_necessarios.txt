
3. Diagramas de Processo:

Use diagramas de processo, como Diagramas de Fluxo de Processo (DFP) ou Diagramas de Atividade, para visualizar o fluxo de trabalho e os processos de negócios. Isso ajuda a identificar atividades, tomadores de decisão e sequência de ações.
4. Diagramas de Caso de Uso:

Use Diagramas de Caso de Uso para representar as interações entre os atores (usuários) e o sistema. Isso ajuda a identificar as funcionalidades e os requisitos do sistema.
5. Diagramas de Classe:

Diagramas de Classe ajudam a representar a estrutura de dados e os objetos no sistema. Eles são úteis para modelar as entidades de negócios, seus atributos e relacionamentos.
6. Diagramas de Sequência:

Diagramas de Sequência são úteis para modelar a interação entre objetos ou componentes em um sistema ao longo do tempo. Eles são especialmente úteis para representar o comportamento de sistemas em resposta a eventos.
7. Diagramas de Estado:

Use Diagramas de Estado para modelar o comportamento dos objetos ou entidades à medida que eles mudam de um estado para outro. Isso é útil para sistemas que têm estados distintos.
